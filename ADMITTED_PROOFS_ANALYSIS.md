## Easy Proofs

These proofs involve straightforward induction, library lemmas, or simple case analysis.

### 1. `expr_step_val_unique` (simp_raven_lang/lang.v)

**Statement**: 
```coq
Lemma expr_step_val_unique e stk_frm v v0:
  expr_step e stk_frm (Val v) ->
  expr_step e stk_frm (Val v0) ->
  v = v0.
```

**Difficulty**: Easy  
**Dependencies**: None  
**Strategy**: 
- Induction on the first `expr_step` derivation
- Each case should force `v0 = v` by examining the structure of `e`
- Key: `expr_step` is deterministic when it produces values

## Key Insights

### Common Patterns
1. **CMRA updates**: Use `auth_update`, `gmap_local_update`, `alloc_local_update`
2. **Expression reasoning**: Induction on structure, case analysis on operators
3. **Translation proofs**: Show commutativity between operations and translation
4. **Type soundness**: Well-typed terms evaluate, evaluation preserves types

### Potential Blockers
1. **Scheme Equality**: May need to understand Coq's code generation better
2. **Fixed-point reasoning**: `stack_free_assertion_trnsl` needs Löb induction
3. **CMRA complexity**: Some admits in `heap_alloc_valid` need epsilon properties

### Tools Needed
- Iris CMRA library (already imported)
- stdpp for sets, maps, lists
- Ltac automation for repetitive cases
- Understanding of `Contractive` and `fixpoint`
